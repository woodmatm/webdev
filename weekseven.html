        import { createStore } from 'redux'

        function todoList(array = [], work) {
            switch (work.type) {
              case 'ADD_TODO':
                return array.concat([work.text])
              default:
                return array
            }
          }
          
          const myStore = createStore(todoList, ['Use Redux'])
          
          myStore.dispatch({
            type: 'ADD_TODO',
            text: 'Read manpage'
          })

This week I found this article which is a great starting point for beginners to React Redux. The reason I decided to take a deeper dive into this article is because Professor Johnson said that next week we will be using Redux in our labs. I wanted to get a head start so that I know what I am doing for our next couple of labs. From the article I learned that, React Redux is the official React UI bindings layer for Redux. It lets your React components read data from a Redux store, and dispatch actions to the store to update state. What I learned is that Redux at the most basic implementation consists of creating a Redux store then subscribing to updates, and then inside the subscription callback you can... get the current store state, extract the data needed by this piece of UI and update the UI with the data. I am glad I looked it up so I can get a head start because it is still a difficult concept to grasp at first. I also know that it closely resembles a Model-View-Controller architecture which I am familiar with from earlier CS courses.
